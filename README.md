# ColorosDrawerHook

ColorosDrawerHook 是一个用于 ColorOS / COUI 桌面应用抽屉分类页的 LSPosed 模块。它可以在模块设置页中图形化调整分类和应用归属，用来替代手动编辑 JSON 配置文件的方式。

> 当前项目主要面向 ColorOS 系统桌面，推荐作用域为 `com.android.launcher`。其他系统或第三方桌面不可用。

## 功能特性

- 图形化管理桌面分类
- 新增、编辑、删除自定义分类
- 查看系统默认分类及其应用
- 为分类勾选已安装应用
- 在应用列表中显示应用图标、包名和当前所属分类
- 按应用名称或包名搜索
- 按分类筛选应用列表
- 支持手动添加包名
- 支持隐藏空分类
- 支持新安装应用默认进入“其他”分类
- 支持 Debug 日志开关
- 支持查看当前 JSON 配置
- 支持备份和恢复配置
- 保存后可自动请求重启桌面，使配置尽快生效

## 环境要求

- Android 9.0 及以上，`minSdk 28`
- 已安装并启用 LSPosed
- ColorOS / COUI 系统桌面
- 推荐作用域：`com.android.launcher`

## 安装与启用

1. 下载并安装模块 APK。
2. 在 LSPosed 管理器中启用模块。
3. 将作用域勾选为系统桌面：`com.android.launcher`。
4. 重启系统桌面，或重启手机。
5. 打开 ColorosDrawerHook 进行配置。

## 使用说明

### 管理分类

打开应用后，一级界面显示分类列表。点击某个分类的“选择应用”进入二级界面，可以为该分类选择应用。

自定义分类会优先显示在列表上方，系统默认分类会显示在后面。

### 选择应用

在分类详情页中，可以通过勾选应用来修改分类归属。已保存到当前分类的应用会被锁定，避免误取消；新勾选但尚未保存的应用仍可取消。

应用列表支持：

- 关键字搜索
- 分类筛选
- 显示应用图标
- 显示当前所属分类
- 显示包名

### 手动添加包名

如果某个应用没有出现在列表中，可以点击右上角的“手动添加”，输入包名后加入当前分类。

### 保存并生效

点击右下角悬浮保存按钮后，会弹出确认窗口。确认后模块会保存配置，并请求重启 ColorOS 桌面。

如果桌面没有立即刷新，可以手动重启桌面或重启手机。

### 备份与恢复

右上角菜单中提供：

- `备份配置`：将当前配置导出为 JSON 文件
- `恢复配置`：从 JSON 文件恢复配置到当前界面

恢复配置后需要再次点击保存并重启，才会写入并生效。

## 项目结构

```text
app/src/main/java/com/nenbucy/colorosdrawerhook/
├── MainActivity.kt                      # 应用设置页入口
├── ConfigStore.kt                       # 共享配置键与常量
├── DrawerConfig.kt                      # 配置数据结构
├── CategorySnapshotContract.kt          # 系统分类快照广播契约
├── SystemCategorySnapshotReceiver.kt    # 接收桌面分类快照
├── model/                               # 分类编辑模型与转换逻辑
├── ui/editor/                           # Compose 设置界面
├── ui/theme/                            # Compose 主题
└── xposed/                              # LSPosed Hook 运行时逻辑
```

## 注意事项

- 本模块会 Hook 系统桌面进程，使用前请确认已了解 LSPosed 模块的基本风险。
- 如果出现桌面崩溃、分类异常或无法进入桌面，可以在 LSPosed 中禁用模块后重启桌面或手机。
- 不同 ColorOS / COUI 版本的桌面实现可能不同，模块不保证所有版本都能正常工作。
- “推荐”分类使用系统自身逻辑处理，本模块不会主动接管推荐分类。
